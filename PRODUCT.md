# Product

## Register

product

## Users

OMG UI 的主要用户是组件库作者本人，以及在其后续 Vue 3 项目中复用组件的开发者。使用场景包括快速搭建产品界面、保持跨项目交互一致，以及在不重复实现基础控件的前提下组合业务页面。

## Product Purpose

OMG UI 是一个仅面向 Vue 3 的个人组件库。它提供边界清晰、类型完整、可访问且可主题化的基础 UI 组件，并通过稳定的命名、子路径导出、文档和测试，让后续项目能够使用同一套视觉与交互语言。成功标准是组件无需业务适配即可复用，API 容易预测，包可以安全发布和独立消费。

## Brand Personality

克制、直接、可靠。界面应当简洁但不简陋，动效用于解释状态和反馈操作，组件在浅色与深色主题中都保持清晰、安静和精确。

## Anti-references

- 不把远程请求、倒计时、权限、路由或状态映射等业务逻辑放进基础组件。
- 不使用复杂装饰、夸张阴影、弹跳动效或过多状态制造视觉噪声。
- 不照搬 Element Plus、Naive UI 或其他框架的外观与 API，只吸收成熟的架构、可访问性和发布实践。
- 不使用多套图标来源、复制 SVG 或创建只转发属性的图标包装层。
- 不覆盖 VitePress 品牌变量，也不让文档站主题泄漏到组件 tokens。
- 不为未上线的旧接口保留兼容别名、私有分支或 CommonJS/UMD 入口。

## Design Principles

1. 组件只表达可复用的视觉和交互，业务含义由使用方组合。
2. 一个概念只保留一个稳定名称、一个类型来源和一个公开入口。
3. 默认行为应当可直接使用，高级能力通过少量正交 props、slots 和 CSS tokens 扩展。
4. 键盘、焦点、ARIA、SSR 和 reduced-motion 与视觉实现同时设计和测试。
5. 先复用共享基础，再新增公开抽象，避免为一次需求过度设计。

## Accessibility & Inclusion

组件以 WCAG 2.1 AA 为最低目标。所有交互组件必须支持完整键盘操作、明确焦点、可访问名称和正确 ARIA 语义；信息不能只依赖颜色表达。文本与交互状态需要满足对比度要求，动画必须提供 `prefers-reduced-motion` 静态替代，并通过单元测试、SSR 测试和 Playwright + axe 验证。
